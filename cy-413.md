# cy-413 心理健康与情绪日记

请根据以下要求生成项目代码和 README。

## 项目主要需求

### 1. 项目概述

**心理健康与情绪日记（MindGarden）** 是一款帮助用户记录情绪变化、进行心理测评、收听冥想音频的个人心理健康应用。系统通过情绪曲线、标签统计、测评报告帮助用户了解自身心理状态。项目类型：**全栈 Web 应用**。

### 2. 核心实体（≥ 3 个，必须贯穿全栈）

- **User（用户）**
  - 字段：id, email, password_hash, nickname, avatar, birth_date, gender, created_at
  - 关键关系：拥有多条 Mood、Assessment、Journal 记录
  - 业务动作：注册/登录、修改资料、查看心理报告
  - **此实体必须贯穿全栈**

- **Mood（情绪记录）**
  - 字段：id, user_id, mood_level（1-10）, mood_tags（焦虑/开心/疲惫/愤怒/平静）, note, record_date, created_at
  - 关键关系：隶属于 User
  - 业务动作：快速记录情绪、按日期筛选、趋势分析
  - **此实体必须贯穿全栈**

- **Assessment（心理测评）**
  - 字段：id, title, description, category（焦虑/抑郁/压力/睡眠）, questions（JSON）, scoring_rule（JSON）
  - 关键关系：用户通过 UserAssessment 参与测评
  - 业务动作：选择测评、答题、生成结果与建议
  - **此实体必须贯穿全栈**

- **Journal（日记）**
  - 字段：id, user_id, title, content, mood_level, weather, is_private, created_at
  - 关键关系：隶属于 User，与 Mood 可联动
  - 业务动作：写日记、编辑、删除、查看历史
  - **此实体必须贯穿全栈**

### 3. 核心页面（≥ 4 个，至少 2 个页面共用同一业务组件）

- **/dashboard（心情花园）**
  - 消费实体：User、Mood、Assessment
  - 关键交互：本周情绪曲线、最新测评推荐、快速记录情绪
  - 共享组件：`<MoodTrendChart>`、`<MoodSelector>`

- **/moods（情绪记录）**
  - 消费实体：Mood
  - 关键交互：情绪日历、列表筛选、新增/编辑情绪
  - 共享组件：`<MoodSelector>`（与 /dashboard 共用）、`<MoodCard>`

- **/assessments（心理测评）**
  - 消费实体：Assessment、User
  - 关键交互：测评列表、答题页面、结果报告
  - 共享组件：`<AssessmentCard>`、`<ResultBadge>`

- **/journals（日记本）**
  - 消费实体：Journal、Mood
  - 关键交互：富文本日记、按心情筛选、时间轴展示
  - 共享组件：`<MoodCard>`（与 /moods 共用）、`<EmptyState>`

- **/profile（个人中心）**
  - 消费实体：User
  - 关键交互：修改资料、查看心理报告汇总
  - 共享组件：`<AvatarUploader>`

### 4. 横切关注点（≥ 2 个）

- **认证授权（JWT + 角色）**
  - 触达文件层：
    - 数据库：`users` 表含 role 字段
    - 后端：`middlewares/auth.js`、`utils/jwt.js`、`constants/roles.js`
    - 路由：写操作需认证，管理员可操作 Assessment
    - 前端：`router/guards.ts`、`stores/authStore.ts`

- **主题切换**
  - 触达文件层：
    - 前端：`stores/themeStore.ts`、`constants/themes.ts`
    - 组件：多个组件读取 theme 状态并切换 class/style
    - 工具：`utils/themeUtils.ts`
    - 样式：多套 CSS 变量定义

- **全局错误处理**
  - 触达文件层：
    - 后端：`middlewares/errorHandler.js`、`utils/AppError.js`
    - 前端：`utils/request.ts` 拦截器、`components/common/GlobalErrorBoundary.tsx`

### 5. 共享枚举/常量（≥ 2 组）

- **MoodTag（情绪标签）**
  - 值：HAPPY = 'happy', ANXIOUS = 'anxious', TIRED = 'tired', ANGRY = 'angry', CALM = 'calm'
  - 前端定义位置：`frontend/src/constants/mood.ts`
  - 后端定义位置：`backend/src/constants/mood.js`
  - 出现位置：Mood 模型、前后端类型定义、MoodSelector、MoodCard、日志模板、错误码
  - **README 必须列出该枚举在前后端的所有出现位置**

- **AssessmentCategory（测评分类）**
  - 值：ANXIETY = 'anxiety', DEPRESSION = 'depression', STRESS = 'stress', SLEEP = 'sleep'
  - 前端定义位置：`frontend/src/constants/assessment.ts`
  - 后端定义位置：`backend/src/constants/assessment.js`
  - 出现位置：Assessment 模型、测评列表页、筛选器、日志模板
  - **README 必须列出该枚举在前后端的所有出现位置**

### 11. 特殊约束：屎山代码设计（低内聚、高耦合、牵一发动全身）

1. **日志模块必须单独管理但全栈引用**
   - 后端必须有一个 `utils/logger.py`，所有 controller/service/middleware 都引用
   - 日志模板集中在 `constants/log_templates.py`，包含至少 20 条模板
   - 任何业务字段变更（如 Mood 新增字段）都必须同步修改日志模板

2. **异常信息必须分散且层层透传**
   - 错误码集中在 `constants/error_codes.py`，但 service 和 controller 都要手动包装
   - 错误 message 必须包含实体名、字段名，如 `"Mood[id=123] update failed: mood_level out of range"`

3. **常量/工具类必须多处耦合**
   - `utils/formatters.py` 同时包含日期格式化、情绪文本、测评分类文本、主题色转换
   - `constants/messages.py` 同时包含前端提示文案、后端返回文案、日志文案
   - 修改一个文案可能同时影响前端显示、后端返回和日志内容

4. **主题切换必须牵一发而动全身**
   - 主题色在 `constants/themes.py`、前端 CSS 变量、ECharts 配置、Ant Design ConfigProvider、utils/formatters.py 中同时存在
   - 新增主题需要修改至少 8 处

5. **枚举必须前后端重复定义且被多处引用**
   - MoodTag 和 AssessmentCategory 必须在前端 constants、后端 constants、类型定义、日志模板、错误码、formatters 中同时存在
   - 新增一个枚举值时，必须修改至少 8 处文件

6. **牵一发动全身的验证标准**
   - 后续如果要求"给 Mood 新增一个 'excited' 标签"，必须触达：数据库 schema、Mood 模型、前后端 constants、类型定义、MoodSelector、MoodCard、moodColor utils、日志模板、错误码、formatters、Dashboard 图表等 ≥ 10 个文件

### 12. 部署要求

这是前后端分离项目。

前端代码放在 `frontend/`：
- 页面、组件、路由、状态管理、API 请求、类型定义、样式和资源等放在对应的子文件夹

后端代码放在 `backend/`：
- 数据库脚本放在 `database/`。

项目必须支持 Docker Compose 一键部署，且在 **任意目录名（包括中文目录名）** 下都能正常启动。

需要提供以下文件：
- 根目录 `docker-compose.yml`
- 根目录 `.env`：包含 `COMPOSE_PROJECT_NAME=mindgarden`
- `frontend/Dockerfile`
- `backend/Dockerfile`
- `frontend/nginx.conf`
- `.env.example`

docker-compose.yml 要求：
- **不要写 `version:` 字段**
- 顶层声明 `name: mindgarden`
- 所有服务使用 `container_name` 命名容器，容器名带 `${COMPOSE_PROJECT_NAME:-mindgarden}` 前缀
- 数据库数据使用命名卷持久化
- 前端通过 Nginx 反向代理后端 API（前端代码里调用 `/api`，由 Nginx 转发到 `backend:<port>`）
- 服务间通过 Docker 内部网络通信
- 数据库服务配置 `healthcheck`，后端服务通过 `depends_on: { db: { condition: service_healthy } }` 等待数据库就绪
- 后端服务也配置 `healthcheck`
- 暴露端口：前端 `18413:80`，后端 `19413:8000`，数据库 `5432:5432`
- 所有可变配置通过环境变量注入
- 支持通过 `docker compose up -d` 一条命令启动

`.env` / `.env.example` 必须包含的键：
- `COMPOSE_PROJECT_NAME=mindgarden`
- `DB_NAME=mindgarden_db`、`DB_USER=mindgarden_user`、`DB_PASSWORD=mindgarden_pwd`
- `DB_ROOT_PASSWORD` 改为 `DB_ADMIN_PASSWORD`
- `JWT_SECRET=change_me_to_a_long_random_string`
- `FRONTEND_PORT=18413`、`BACKEND_PORT=19413`、`DB_PORT=5432`

Nginx 反向代理要求（`frontend/nginx.conf`）：
- `location /api/` 反向代理到 `http://backend:8000/`
- 配置 `try_files $uri $uri/ /index.html;` 支持前端 SPA 路由
